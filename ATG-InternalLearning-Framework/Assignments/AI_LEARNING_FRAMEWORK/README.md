## Assignment Structure

Each assignment follows the same one-page template with:

### Standard Sections
1. **Assignment Information** - Category, subcategory, team members, date, duration
2. **Learning Objectives** - 3 clear objectives
3. **Topics to Cover** - 4 main topics
4. **Assignment Deliverables**
   - Presentation (20-30 min)
   - Code Demo/Example
   - Resources & References (3-5 curated)
   - Q&A Session (10-15 min)
5. **Preparation Guidelines** - Research, practice, collaboration, engagement tips
6. **Success Criteria** - Checklist of 5 success markers
7. **Support & Resources** - References and contacts
8. **Notes** - Additional context and connections

## Assignment Scheduling Guidelines

### Weekly Cadence
- **1 session per week:** 43 weeks total (~10 months)
- **2 sessions per week:** 22 weeks total (~5 months)
- **Round-robin format:** Rotate through team pairs

### Team Size Considerations
- **8 team members:** 4 pairs, each presents ~10-11 times
- **10 team members:** 5 pairs, each presents ~8-9 times
- **12 team members:** 6 pairs, each presents ~7-8 times

### Suggested Schedule
- **Months 1-3:** Categories 1-4 (Foundations + RAG)
- **Months 4-6:** Categories 5-7 (Agents + Application + MLOps)
- **Months 7-9:** Categories 8-10 (Data + SE + Advanced)
- **Month 10:** Categories 11-12 (Applications + Business)
