## Status Legend

- **Not Started** - Assignment not yet assigned
- **Assigned** - Team pair assigned, preparation in progress
- **Draft Ready** - Materials ready for review
- **Scheduled** - Date set for presentation
- **Completed** - Presentation delivered, materials shared
- **Archived** - Recording and materials in knowledge base

## Metrics to Track

### Preparation Phase
- Days to prepare after assignment
- Draft submission on time (Y/N)
- Code demos tested (Y/N)

### Presentation Phase
- Attendance count
- On-time start (Y/N)
- Duration within limits (Y/N)
- Q&A engagement (1-5 scale)

### Post-Presentation
- Materials shared within 24h (Y/N)
- Recording uploaded (Y/N)
- Team satisfaction (1-5 scale)
- Knowledge applied to projects (Y/N)

## Notes

- Update this tracking sheet weekly
- Rotate team pairs to ensure equal distribution
- Allow 1-2 weeks preparation time per assignment
- Schedule lighter weeks around holidays/busy project periods
- Consider 2 presentations per week if team is large
- Keep recordings and materials in shared knowledge base
