## Learning Objectives
By the end of this session, the team should be able to:
1. Apply the principle of least privilege in code
2. Implement defense in depth strategies
3. Conduct security code reviews effectively
4. Manage secrets and credentials securely

---

## Topics to Cover

### 1. Principle of Least Privilege
- Minimal permissions for users and services
- Service accounts and API keys
- Database user permissions
- File system permissions
- Temporary privilege elevation

### 2. Defense in Depth
- Multiple layers of security
- Redundant controls
- Fail-secure vs fail-open
- Security boundaries
- Assume breach mentality

### 3. Security Code Review Checklist
- Authentication and authorization
- Input validation
- Error handling and logging
- Cryptography usage
- Third-party dependencies

### 4. Secrets Management
- Never commit secrets to code
- Environment variables vs secret managers
- Secret rotation
- Access control for secrets
- Tools: HashiCorp Vault, AWS Secrets Manager, Azure Key Vault
