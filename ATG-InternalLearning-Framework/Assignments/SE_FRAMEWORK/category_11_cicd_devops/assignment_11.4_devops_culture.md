# Assignment 11.4: DevOps Culture

## Assignment Information
- **Category**: CI/CD & DevOps
- **Sub-category**: 11.4 DevOps Culture
- **Team Members**: [Pair names to be assigned]

---

## Learning Objectives
By the end of this session, the team should be able to:
1. Understand DevOps principles and philosophy
2. Foster collaboration between development and operations
3. Adopt an automation mindset
4. Practice continuous improvement

---

## Topics to Cover

### 1. DevOps Principles
- CALMS framework (Culture, Automation, Lean, Measurement, Sharing)
- Break down silos
- Shared responsibility model
- You build it, you run it
- Blameless post-mortems
- Continuous learning

### 2. Collaboration Between Dev and Ops
- Cross-functional teams
- Shared goals and metrics
- On-call rotation including developers
- Empathy and understanding
- Joint problem-solving
- Communication practices

### 3. Automation Mindset
- Automate repetitive tasks
- Toil reduction
- Self-service platforms
- Infrastructure automation
- Testing automation
- Documentation automation

### 4. Continuous Improvement
- Kaizen and retrospectives
- Metrics-driven improvement
- Experimentation and learning
- Fail fast, learn quickly
- Technical debt management
- Innovation time
