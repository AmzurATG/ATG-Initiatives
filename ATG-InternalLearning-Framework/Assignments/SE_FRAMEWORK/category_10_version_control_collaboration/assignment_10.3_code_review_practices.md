# Assignment 10.3: Code Review Practices

## Assignment Information
- **Category**: Version Control & Collaboration
- **Sub-category**: 10.3 Code Review Practices
- **Team Members**: [Pair names to be assigned]
- **Presentation Date**: [To be scheduled]
- **Duration**: 30-45 minutes

---

## Learning Objectives
By the end of this session, the team should be able to:
1. Conduct effective code reviews following established guidelines
2. Use review checklists and maintain consistent standards
3. Give and receive constructive feedback
4. Manage pull requests effectively

---

## Topics to Cover

### 1. Effective Code Review Guidelines
- Review for correctness, design, testing
- Code readability and maintainability
- Performance and security considerations
- Consistent style and conventions
- Documentation completeness
- Reviewer assignment strategies

### 2. Review Checklist and Standards
- Functionality and logic
- Error handling
- Test coverage
- Security vulnerabilities
- Performance implications
- Naming and style
- Documentation

### 3. Giving and Receiving Feedback
- Constructive vs critical feedback
- Specific, actionable comments
- Asking questions vs demanding changes
- Acknowledging good work
- Handling disagreements
- Learning from reviews

### 4. Pull Request Best Practices
- PR size and scope
- Clear descriptions and context
- Linking to issues
- Self-review before requesting
- Responding to comments
- Timely reviews
