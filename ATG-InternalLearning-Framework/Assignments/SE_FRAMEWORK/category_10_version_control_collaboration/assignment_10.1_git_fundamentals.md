## Learning Objectives
By the end of this session, the team should be able to:
1. Master Git workflow basics and commands
2. Understand branching and merging strategies
3. Write effective commit messages
4. Apply Git best practices consistently

---

## Topics to Cover

### 1. Git Workflow Basics
- Initialize, clone, and configure repositories
- Staging area concept
- Commit, push, pull, fetch operations
- Working with remotes
- Viewing history and diffs
- Undoing changes safely

### 2. Branching and Merging
- Creating and switching branches
- Merge vs rebase
- Merge conflicts resolution
- Fast-forward merges
- Cherry-picking commits
- Branch cleanup

### 3. Commit Message Conventions
- Conventional commits format
- Subject line best practices
- Body and footer sections
- Referencing issues
- Atomic commits
- Commit message templates

### 4. Git Best Practices
- Commit frequently, push often
- Keep commits atomic and focused
- Never commit secrets or sensitive data
- .gitignore patterns
- Git hooks for automation
- Repository hygiene
