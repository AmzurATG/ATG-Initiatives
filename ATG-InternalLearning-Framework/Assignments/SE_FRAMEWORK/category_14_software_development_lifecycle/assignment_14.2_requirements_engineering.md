# Assignment 14.2: Requirements Engineering

## Assignment Information
- **Category**: Software Development Lifecycle
- **Sub-category**: 14.2 Requirements Engineering
- **Team Members**: [Pair names to be assigned]
- **Presentation Date**: [To be scheduled]
- **Duration**: 30-45 minutes

---

## Learning Objectives
By the end of this session, the team should be able to:
1. Gather and elicit requirements effectively
2. Write clear user stories and acceptance criteria
3. Manage changing requirements
4. Validate requirements with stakeholders

---

## Topics to Cover

### 1. Requirements Gathering
- Stakeholder identification
- Elicitation techniques (interviews, workshops)
- Requirements discovery
- Functional vs non-functional requirements
- User research methods
- Domain understanding

### 2. User Stories and Acceptance Criteria
- User story format (As a... I want... So that...)
- INVEST criteria (Independent, Negotiable, Valuable, Estimable, Small, Testable)
- Acceptance criteria writing
- Given-When-Then format
- Story splitting techniques
- Definition of Done

### 3. Requirements Management
- Prioritization techniques (MoSCoW, WSJF)
- Backlog management
- Handling changing requirements
- Traceability
- Requirements versioning
- Conflict resolution

### 4. Requirements Validation
- Stakeholder reviews
- Prototyping and mockups
- Requirements walkthrough
- Test-driven requirements
- Feedback loops
- Sign-off processes
