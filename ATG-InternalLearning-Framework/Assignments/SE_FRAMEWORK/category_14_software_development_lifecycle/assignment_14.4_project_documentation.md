## Learning Objectives
By the end of this session, the team should be able to:
1. Create appropriate project documentation
2. Balance documentation with agility
3. Maintain living documentation
4. Use documentation tools effectively

---

## Topics to Cover

### 1. Documentation Types
- Architecture Decision Records (ADRs)
- README files
- API documentation
- Runbooks and playbooks
- Project wikis
- Diagram and visual documentation

### 2. Documentation Balance
- Just enough documentation
- Working software over documentation
- Living documentation principles
- Documentation as code
- When to document
- Avoiding documentation debt

### 3. Maintaining Documentation
- Continuous documentation updates
- Documentation in Definition of Done
- Automated documentation generation
- Documentation reviews
- Deprecation and archival
- Knowledge transfer

### 4. Documentation Tools
- Markdown for technical docs
- Confluence/Notion for wikis
- Swagger/OpenAPI for APIs
- Diagram tools (Mermaid, PlantUML)
- GitHub wikis
- Documentation site generators (MkDocs, Docusaurus)
