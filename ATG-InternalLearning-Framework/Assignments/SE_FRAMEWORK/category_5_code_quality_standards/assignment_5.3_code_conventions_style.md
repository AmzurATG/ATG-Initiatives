# Assignment 5.3: Code Conventions & Style

## Assignment Information
- **Category**: Code Quality & Standards
- **Sub-category**: 5.3 Code Conventions & Style
- **Team Members**: [Pair names to be assigned]
- **Presentation Date**: [To be scheduled]
- **Duration**: 30-45 minutes

---

## Learning Objectives
By the end of this session, the team should be able to:
1. Follow language-specific style guides consistently
2. Establish and maintain team coding standards
3. Use code formatting tools for consistency
4. Write effective documentation and comments

---

## Topics to Cover

### 1. Language-Specific Style Guides
- JavaScript: Airbnb, StandardJS, Google
- Python: PEP 8, Black style
- Java: Google Java Style, Oracle conventions
- C#: Microsoft conventions
- Understanding rationale behind style choices

### 2. Team Coding Standards
- Establishing team conventions
- Documentation and enforcement
- Consistency across projects
- Evolving standards over time
- Code review for standards compliance

### 3. Code Formatting and Consistency
- Indentation and spacing
- Line length and wrapping
- Import organization
- File structure conventions
- Automated formatting tools

### 4. Documentation Standards
- When to comment and when not to
- JSDoc, Javadoc, Python docstrings
- README documentation
- API documentation
- Architecture decision records (ADRs)

- Overview of popular style guides

### 3. Resources & References (Provide 3-5 curated)

- Documentation maintenance
