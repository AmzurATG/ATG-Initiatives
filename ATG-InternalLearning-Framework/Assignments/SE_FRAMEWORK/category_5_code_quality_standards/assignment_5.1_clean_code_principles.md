## Assignment Information

## Learning Objectives
By the end of this session, the team should be able to:
1. Apply meaningful naming conventions and improve code readability
2. Design functions and methods that do one thing well
3. Organize code structure for maintainability using DRY, KISS, and YAGNI principles

---

## Topics to Cover

### 1. Naming Conventions and Readability
- Meaningful variable and function names
- Avoiding abbreviations and single letters
- Consistency across codebase
- Self-documenting code

### 2. Function and Method Design
- Single responsibility for functions
- Function size and complexity
- Parameter count guidelines
- Return value clarity

### 3. Code Organization and Structure
- File and folder organization
- Class and module structure
- Separation of concerns
- Cohesion and coupling

### 4. DRY, KISS, YAGNI Principles
- Don't Repeat Yourself (DRY)
- Keep It Simple, Stupid (KISS)
- You Aren't Gonna Need It (YAGNI)
- Balance and trade-offs
