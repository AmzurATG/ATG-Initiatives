What are "Dimensions"?

Simple Analogy:

  • 2D Vector: [x, y] = Point on a map (2 dimensions)
  • 3D Vector: [x, y, z] = Point in space (3 dimensions)
  • Embedding Vector: [d‚ÇÅ, d‚ÇÇ, ..., d‚ÇÅ‚ÇÖ‚ÇÉ‚ÇÜ] = Point in semantic space (1536 dimensions)

What each dimension represents:

  • Dimension 1: Might capture "technical vs casual"
  • Dimension 2: Might capture "positive vs negative sentiment"
  • Dimension 3: Might capture "abstract vs concrete"
  • ... and so on for all 1536 dimensions

How to Choose Dimension Size?

  1. Start Small (384-768): For development and testing
  2. Use Medium (1024-1536): For production with good quality
  3. Go Large (3072): Only if accuracy is critical and you have budget
  4. Consider: Storage costs, query speed requirements, accuracy needs

Each dimension captures different semantic patterns - more dimensions = richer understanding but higher costs

What is Cosine Similarity?

Measures the angle between two vectors to determine semantic similarity. Ranges from -1 (opposite) to 1 (identical).

import numpy as np def cosine_similarity(vec_a, vec_b): dot_product = np.dot(vec_a, vec_b) norm_a = np.linalg.norm(vec_a) norm_b = np.linalg.norm(vec_b) return dot_product / (norm_a * norm_b) # Example usage vec_ml = [0.23, -0.89, 0.44, ...] # "machine learning" vec_ai = [0.25, -0.87, 0.43, ...] # "artificial intelligence" similarity = cosine_similarity(vec_ml, vec_ai) # Result: 0.92 (highly similar!)

8. Implementation Checklist

  • Choose Embedding Model
    Recommended: OpenAI text-embedding-3-small (1536 dims)

    Install extension, create vector column, add index
  • Implement Chunking
    Target: 500-1000 tokens, 10-20% overlap
  • Build Embedding Pipeline
    Batch processing, rate limit handling, metadata storage
  • Create Similarity Search
    Cosine distance, top-k=3-5, include scores
  • Optimize Performance
    Add IVFFlat/HNSW index, implement caching
  • Test & Monitor
    Measure query times, validate results quality
